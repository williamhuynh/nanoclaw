# AiD Agent Delivery Positioning — "Tool Composition, Not App Build"

## The Finding
AiD's positioning for agent engagements is:

> **We don't structurally build applications. We incorporate the right client tools into agents.**

The delivery model is tool-composition on top of a client's existing software stack — agents orchestrate the tools the client already has (and any purpose-built connectors), rather than AiD building net-new applications.

## Why it matters
- Keeps AiD out of the app-dev / systems-integrator lane (wrong cost base, wrong capability fit).
- Makes pricing cleaner: the "build" is the agent + tool integration, not a bespoke application project.
- Aligns with the AI Studio offering (strategy + targeted delivery) rather than a software house.
